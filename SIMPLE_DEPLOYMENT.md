## Simple Deployment Options

### Option 1: Direct Copy-Paste to Wiki (Simplest)

1. Open `index.html` in a text editor
2. Copy the entire file content (Ctrl+A, Ctrl+C or Cmd+A, Cmd+C)
3. Go to your Wiki page
4. Look for an "Edit" or "HTML" mode
5. Paste the entire HTML content
6. Save the page

**Note**: Some Wikis have an "HTML" or "Source" mode where you can paste raw HTML.

### Option 2: Upload HTML File

If your Wiki supports file uploads:

1. Upload `index.html` to the Wiki
2. Link to it or embed it in a Wiki page

### Option 3: Use Wiki HTML Widget/Component

Many Wikis have HTML widgets or components:

1. Find the HTML/Widget component in Wiki editor
2. Paste your HTML content there
3. Save

### Option 4: GitHub Pages (External, but Simple)

1. Go to: https://github.com/vamsi676/t1-t2-mapper
2. Settings → Pages
3. Source: `main` branch, `/ (root)`
4. Save
5. Your app will be live at: `https://vamsi676.github.io/t1-t2-mapper/`

Then you can share this URL or embed it in Wiki using an iframe.

### Option 5: Internal Web Server

If you have access to an internal web server:

1. Upload `index.html` to the web server
2. Access it via URL
3. Share the URL or embed in Wiki

## Recommended: Direct Copy-Paste

**The simplest approach is Option 1** - just copy and paste your HTML directly into the Wiki page if it supports HTML editing.

## Testing Your HTML File

Before deploying, test locally:

1. Open `index.html` in a web browser
2. Verify everything works
3. Then deploy using one of the methods above
